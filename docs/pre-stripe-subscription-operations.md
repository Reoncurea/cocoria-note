# Stripe導入前のサブスク仮運用ルール

## 基本方針

Stripeを実装するまでは、課金状態は管理者画面で手動管理する。
利用者の識別はメールアドレスではなく `user_id` を基準にする。

## 利用状態の意味

| 状態 | 意味 | 利用可否 |
| --- | --- | --- |
| 試用中 | 初回利用開始から1か月の無料試用期間 | 利用可 |
| 有効 | 支払い確認済み、または継続利用を許可済み | 利用可 |
| 支払い確認 | 試用期間終了、未入金、支払方法確認など | 利用不可 |
| 停止中 | 解約・停止・利用終了 | 利用不可 |

## 試用期間

- 初回パスワード設定完了時を利用開始日とする。
- 試用終了日は利用開始日から1か月後。
- 試用終了日を過ぎると、自動的にアプリ利用は止まり、支払い確認画面が表示される。
- 管理者画面を開くと、期限切れの試用中ユーザーは `支払い確認` に同期される。

## 手動請求の流れ

1. 管理者画面で `試用終了間近` または `支払い確認` の利用者を確認する。
2. 利用者に継続意思と支払方法を確認する。
3. 入金または継続許可が確認できたら、管理者画面で利用状態を `有効` にする。
4. `有効` に変更すると、次回更新予定日は操作日から1か月後になる。
5. 継続しない場合は `停止中` にする。

## Stripe導入時に追加するもの

- `stripe_customer_id`
- `stripe_subscription_id`
- Stripe Webhookで `subscription_status` を自動更新する処理
- 決済失敗時の `支払い確認` への自動変更
- 解約時の `停止中` への自動変更

## 注意

メールアドレスは変更される可能性があるため、課金や権限の基準にはしない。
メールアドレスは表示・連絡用、`user_id` は本人識別用として扱う。
